# Code of Conduct

`tensor` is an educational project. We expect everyone interacting in this repository — issues, pull requests, discussions, tutorial feedback — to follow a short list of professional norms:

- **Be respectful** in tone and word choice.
- **Assume good faith.** Most disagreements are caused by missing context, not bad intent.
- **Stay on the project.** Reviews and discussions focus on the code, the design, the documentation, or the tutorial — not on the person.
- **Welcome learners.** This is an educational artifact. Beginner-level questions are exactly the kind of feedback the project wants.
- **Respect the solo-maintainer bandwidth.** Fast responses are not guaranteed. Patience is appreciated; nagging is not.

## Scope

This applies anywhere project activity happens: the GitHub repository, the Jupyter Book site, the tutorial notebooks, and any communication explicitly framed as `tensor`-related.

## Enforcement

If you experience or observe behavior that breaks the norms above, contact the maintainer privately. Reports are read, considered, and acted on at the maintainer's discretion. Possible outcomes include a private clarification, a public clarification on the relevant issue or PR, an edit or removal of the offending content, or a block of the involved account from the repository.

Anyone is welcome to ask for a clarification of any decision; the maintainer's choice on a given case is final but not precedent-binding for future cases.

## A note on what this isn't

This document is intentionally short. It is not a substitute for thinking; it is a short pointer toward the kind of project we want this to be. If a situation arises that the norms above don't obviously cover, the maintainer will use judgment. Standard professional behavior — the kind you'd extend to a colleague you respect — is the working assumption.
