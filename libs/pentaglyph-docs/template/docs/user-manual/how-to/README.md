# how-to — problem-oriented (Diátaxis)

> Authoritative source: <https://diataxis.fr/how-to-guides/>

A how-to guide solves **one specific problem** for a **competent reader**. The reader knows the basics and arrives with a goal in mind.

## Rules

1. **Start from a problem, not a topic.** Title format: "How to <verb> <object>".
2. **Assume competence.** Skip introductory material; link to tutorials for newcomers.
3. **Cover relevant alternatives.** Unlike tutorials, how-to guides discuss options.
4. **One problem per file.** Don't bundle.

## Suggested structure (per file)

1. **When to use this guide** — one sentence stating the trigger.
2. **Prerequisites** — what the reader must already have set up.
3. **Steps** — sequenced, but the reader may adapt them.
4. **Variations / alternatives** — when applicable.
5. **Related** — links to reference and explanation.
