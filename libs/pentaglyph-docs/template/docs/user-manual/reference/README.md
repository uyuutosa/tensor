# reference — information-oriented (Diátaxis)

> Authoritative source: <https://diataxis.fr/reference/>

Reference is **austere, complete, structured**. Reader looks up a fact, leaves. No narrative.

## Rules

1. **No narrative voice.** Just facts. Tables, lists, definitions.
2. **Complete coverage of one well-bounded surface** — e.g. all CLI flags, all API endpoints, all config keys.
3. **Generated when possible.** If the schema is in code, generate the reference; do not hand-maintain.
4. **No examples that tell stories.** Examples are minimal usage demonstrations only. Stories belong in tutorials.

## Suggested structure (per file)

- A single table or alphabetised list.
- Each row: name, type, default, description, example.
- A "see also" footer linking to related reference pages.
