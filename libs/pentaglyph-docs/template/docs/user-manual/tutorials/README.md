# tutorials — learning-oriented (Diátaxis)

> Authoritative source: <https://diataxis.fr/tutorials/>

A tutorial is a **guaranteed-to-succeed first-run experience**. The reader is a newcomer; the goal is for them to feel "I made it work and I want to learn more".

## Rules

1. **Show, do not explain.** Reserve "why" for the explanation quadrant.
2. **Guaranteed success.** Test every step on a clean environment. If a step can fail, fix the tutorial, not the reader.
3. **Hide trade-offs.** Tutorials make choices for the reader; alternatives belong in how-to or reference.
4. **One topic per file.** If you need two tutorials, write two files.

## Suggested structure (per file)

1. **What you'll build** — one paragraph showing the end state.
2. **Prerequisites** — software, accounts, knowledge.
3. **Steps** — numbered, copy-pasteable, no skipping.
4. **What you learned** — recap.
5. **Next steps** — links to related how-to and explanation pages.
