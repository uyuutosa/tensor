# explanation — understanding-oriented (Diátaxis)

> Authoritative source: <https://diataxis.fr/explanation/>

Explanation is **discursive, theoretical, deepening**. Reader sits down to understand *why* something is the way it is. No instructions.

## Rules

1. **Discursive voice.** Like an essay, not a recipe.
2. **Background, context, history, alternatives, trade-offs.** This is where the "why" lives.
3. **No step-by-step instructions.** If you find yourself writing "first, do X", you are writing a how-to in the wrong quadrant.
4. **Link to ADRs**: explanation pages often summarise the human-readable narrative behind one or more ADRs in `arc42/09-decisions/`.

## Suggested structure (per file)

1. **What this is about** — one paragraph.
2. **Background** — the situation before the design existed.
3. **The design** — what we built and how it works conceptually.
4. **Alternatives we considered and rejected** — and why.
5. **Trade-offs we accept** — honest.
6. **Related** — ADRs, reference pages.
