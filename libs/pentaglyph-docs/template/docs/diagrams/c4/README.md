# C4 model diagrams

> Authoritative source: <https://c4model.com>
>
> Recommended toolchain: **Structurizr DSL** — <https://docs.structurizr.com/dsl> — as the canonical workspace; rendered SVG exports are **committed** so the diagrams are browseable from the repo web UI (GitHub / CodeCommit / Bitbucket) without local tooling.

| File             | Purpose                                                                                              |
| ---------------- | ---------------------------------------------------------------------------------------------------- |
| `workspace.dsl`  | Single source of truth for L1–L3 (system context, containers, components)                            |
| `exports/`       | Rendered `.svg` output. **Committed**, regenerated from `workspace.dsl` via `/diagram-render`        |
| `views.md`       | Auto-generated viewer that embeds every SVG under `exports/` (regenerated by the skill)              |

## Levels

- **L1 — System Context**: who uses the system, what external systems it touches.
- **L2 — Container**: high-level technology choices (apps, services, datastores).
- **L3 — Component**: components inside each container.
- **L4 — Code**: only when needed; usually inline in arc42 §5 building-block notes.

## How to render

```bash
# Canonical path — regenerates SVGs and views.md
/diagram-render

# Manual fallback (one-off, no viewer-md update)
structurizr-cli export -workspace workspace.dsl -format plantuml -output exports/
plantuml -tsvg exports/*.puml
```

The `/diagram-render` skill ([`.claude/skills/diagram-render/SKILL.md`](../../../.claude/skills/diagram-render/SKILL.md)) is the canonical regenerator. It walks the diagram tree, picks the right CLI per notation (Structurizr / PlantUML / Mermaid), produces deterministic SVGs, and refreshes `views.md` automatically. Run before commit; CI may run `/diagram-render --check` to fail PRs whose SVGs drift from sources.

## Hard rules

1. **`workspace.dsl` is the only source of truth.** Hand-drawn diagrams that diverge from it are bugs.
2. **`exports/` is committed.** Regenerate via `/diagram-render`; CI may enforce `--check` to block drift. The historical "exports/ gitignored" rule is replaced because CodeCommit / corporate Git web UIs do not render Mermaid in markdown — committed SVGs are the only way to make architecture browseable from the repo UI without local tooling.
3. **Element names** in `workspace.dsl` must match exactly the names used in `arc42/05-building-blocks/` and `detailed-design/`.
