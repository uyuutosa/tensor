> This section answers: *what is the system, who does it serve, and what does success look like?*

## What lives here

| Suggested file | Purpose |
| -------------- | ------- |
| `overview.md`  | Scope, primary stakeholders, top-level goals (G-1, G-2, …), success criteria, quality goals |

## How to write here

1. Pick template [`../../templates/1_architecture-overview.md`](../../templates/1_architecture-overview.md) (§1–§5+§8 multi-section template) or split into a focused per-section file.
2. State the system's identity, the problems it solves, the stakeholders it serves, and the **measurable** outcomes that define success.
3. The content must be self-contained: a reader who has never seen any other doc in this repo should be able to understand the system's purpose, value, and design intent from this section alone.

For lifecycle / when to update, see [`../../WORKFLOW.md`](../../WORKFLOW.md).
