## What lives here

| Suggested file | Purpose                                                                 |
| -------------- | ----------------------------------------------------------------------- |
| `glossary.md`  | Single alphabetised table of project-specific terms and abbreviations   |

## How to write here

1. **Add a term on first use** in any other doc. If you find yourself defining the same term twice, move the definition here and link.
2. Each entry: *term* / *definition* / *reference* (link to authoritative source if external).
3. Avoid restating widely-known industry terms — only entries that are *project-specific* or *industry-specific in a non-obvious way*.
4. Sort alphabetically. Group abbreviations and full terms in the same table; reference each from the other.

For lifecycle / when to update, see [`../../WORKFLOW.md`](../../WORKFLOW.md).
