# lyx-export

LyX → `tensor::tex` translator. Closes the LyX half of [ADR-0005](../docs/arc42/09-decisions/0005-adopt-tex-lyx-as-authoring-surface.md): a learner authors math in LyX (WYSIWYG), this tool extracts every Formula inset and emits a C++ snippet that wraps each formula in the `_tex` UDL.

The MVP recognises inline (`$...$`) and display (`\[ ... \]`) math insets. Richer LyX features (LyX-Code blocks, ERT, custom modules) are pass-through.

## Usage

```bash
python lyx-export/lyx_to_tex.py path/to/notes.lyx
# or write to a file
python lyx-export/lyx_to_tex.py path/to/notes.lyx --output notes.cpp
```

Output (excerpt):

```cpp
// Generated by lyx_to_tex.py from a LyX document.
// Each block below corresponds to one Formula inset.
#include <tensor/tex/tex.hpp>

using namespace tensor::tex::literals;

// Formula #1 (LyX line 12):
auto formula_1 = R"(c_{ij} = a_i + b_j)"_tex;

// Formula #2 (LyX line 18):
auto formula_2 = R"(\sum_i {a_i b_i})"_tex;
```

The generated symbols are `tensor::tex::Expression` instances; feed them to `tensor::tex::Evaluator` with named bindings (see [`tutorials/05_autograd-from-scratch.ipynb`](../tutorials/05_autograd-from-scratch.ipynb) for the bind-then-evaluate pattern).

## Use it as a LyX plugin (recommended)

The CLI works standalone, but the natural integration is to register `lyx_to_tex.py` as a LyX **export converter** so the user gets a **File → Export → Tensor C++** menu entry. See [`LYX_PLUGIN.md`](./LYX_PLUGIN.md) for the full instructions. One-shot helper:

```bash
./lyx-export/install_lyx_converter.sh
```

The helper detects the OS, appends two lines to the LyX preferences file, and prints a "restart LyX" reminder. Optional: also drop [`tensor-math.module`](./tensor-math.module) into LyX's user layouts directory to get a labelled `Flex:TensorMath` inset.

## Round-trip

The MVP is one-way (LyX → C++). The reverse (C++ AST → LyX inset) is feasible — `tensor::tex::to_latex` already produces a clean LaTeX string — but is not implemented here yet.

## Limitations

- Only Formula insets are recognised. LyX math macros (`\macro` declarations) and array environments outside of `\sum`-style constructs may not survive the round trip through `tensor::tex::parse`.
- The `_tex` parser supports the LaTeX subset documented in [`include/tensor/tex/parser.hpp`](../include/tensor/tex/parser.hpp); inputs outside that subset will produce a `std::runtime_error` at parse time. Run a quick smoke test before committing the emitted `.cpp`.
